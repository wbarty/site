Fourier-Based Option Pricing (FFT) Theory/Python Application

Posted on 2020-09-03

First off, why use Fourier Transforms for option pricing? The algorithm can be applied to any problem when the characteristic function of the driving process of uncertainty is known, in finance this is very common making it a good choice. This is a semi-analytical formula, it will need to be computed numerically however the computational cost is low for the degree of accuracy, especially when compared to something like Monte Carlo Simulation which is computationally expensive. Finally, FFT or Fast-Fourier-Transforms are quick as the name might suggest. For options pricing, model calibration to option quotes and implied volatilities are all areas where FFT excels, the following will give an overview of what it is and then some examples in Python, unfortunately, QuantLib doesn’t have a FFT engine in Python however there is one available in the C++ library.

Fourier Transforms

Complex Numbers

We have to start with a brief introduction to complex numbers in order to understand how the Fourier Transform (FT) works. Essentially the FT is all about evenly spaced points on a circle, the best way to describe the location of such points it with complex numbers. These numbers allow one to describe a vector in a 2-dimensional space, the following shows such a vector of 2+i:

The point have moved up to 2 on the Real axis and up 1i on the Imaginary (complex) axis.

The addition, subtraction and multiplication of a vectors of complex numbers works the same as a regular vector of ordinary numbers, multiplying by a scalar as well. Now it is worth pointing out that i representing an imaginary number, is the square root of -1, this works in reverse also and i-squared is equal to -1.

The following will get deeper into geometry and is referenced from Ales Cerny’s paper but is crucial to understanding the whole point of the FT.

Complex numbers can be used to describe the movement of a point around a circle, the circle in the next figure intersects the real axis at (1,-1) and imaginary axis at (i,-i).

The line OA and specifically the point A is specified by the angle phi, this is its argument and determines where it meets the circle. We can show the location of point A using

cosϕ+isinϕ

Generally, when dealing with sin and cos you’ll need to use radian rather than angle degrees, to convert units:

Radians=(π180∘)⋅degrees
Degrees=(180∘π)⋅radians

Complex number properties:

If i=90∘ then i⋅i=180∘, the imaginary number angles on
unit circles are additive when multiplied,
one may note that 180∘ on the unit circle (seen above with unit 1)
corresponds to -1 which is inline with that fact
that −1=i and therefore i⋅i=i2=−1

Using this rule, complex multiplication follows:
(a+ib)(x+iy)=ax+by(i2)+i(bx+ay)

The unit circle can also be represented with the Euler Formula, for example:
cosϕ+isinϕ=eiϕ
and therefore with the additivity of angles:
eiϕ1⋅eiϕ2=ei(ϕ1+ϕ2)

Now using all this we can pretty easily add evenly spaced spokes onto a wheel represented by our unit circle.

If we want 5 spokes then we can simply divide the entire circle by 5, i.e. 2π5
The initial spoke will be placed at point ei2π5
This point will be denoted z5 and it known as the 5th root of unity.
When working with the circle we move anti-clockwise so each time we multiply z5
by itself we rotate the circle by 2π5 or one sixth and add a new spoke.
Since the multiplications are additive (z5)2,(z5)3,…,(z5)5 completes the circle,
where (z5)5=(z5)0=1 which is the start of the circle, furthermore, every 5 rotations
are equal, e.g. (z5)0=(z5)5=(z5)10 and so on. This is also true if we went in reverse
and rotated clockwise, with (z5)0=(z5)−5=(z5)−10 which are equal to the anti-clockwise counterparts.

Using the above we can note that for a given sequence of numbers such as a=[a0,a1,…,an−1]
the reverse is rev(a)=[a0,an−1,…,a1] making note that rev(a) starts with a0 just like sequence a.

Therefore, for any k the sequence: (znk)0,(znk)1,…,(znk)n−1 is identical to (zn−k)0,(zn−k)1,…,(zn−k)n−1 in reverse order

Due to the additivity this works because, (zn−k)n−j=zn−kn+nj=znkj=(znk)j for any value of j.
This is the basis for Discrete Fourier Transform.

Discrete Fourier Transform (DFT)

Using zn≡ei2πn which again is the nth root of unity, along with our sequence a0,a1,…,an−1 which are complex numbers (in general). The DFT of the sequence a0,a1,…,an−1 is the sequence b0,b1,…,bn−1 such that the following holds:
bk=a0(znk)0+a1(znk)1+⋯+an−1(znk)n−1n=1n∑j=0n−1ajznjk, and since zn≡ei2πn,bk=1n∑j=0n−1ajei2πnjk

For simplicity: F=b
This is the forward transform, there is also the inverse transform. The Inverse DFT is the same as the forward DFT of the same sequence but in reversed order:

a~l=b~0(zn−l)0+⋯+b~n−1(zn−l)n−1n=1n∑k=0n−1b~kzn−kl=1n∑k=0n−1b~ke−i2πnkl
or a~=F−1(b~)

The inverse DFT with sequence b~0,…,b~n−1 is the same as the forward DFT of the same sequence in reversed order giving:

F−1(b~)=F(rev(b~)), this works either way, inverting the transform of one gives the forward transform of the sequence.

Continuous Fourier Transform

For financial applications in general we work in continuous time for its ease of use, FT is no different, that being said, Ales Cerny’s paper describes using DFT for binomial option pricing if you are interested, we are going to skip over that for now, however as usual we need the discrete version for computation.

The Fourier Transform of the integrable function f(x) is given by:
f~(u)≡∫−∞∞eiuxf(x)dx, u can be either real or complex, the term eiux is called the phase factor

Fourier Inversion gives:
f(x)=12π∫−∞∞e−iuxf~(u)du for real values of u and,
f(x)=12π∫−∞+iui∞+iuie−iuxf~(u)du for complex values of u.

u=ur+iu where latexur is a real part of u and iu is an imaginary part of u hence why iu is added to the integral.

Next lets describe the characteristic function, let X be a real random variable, if it has a probability density function (such as the Normal PDF), then the characteristic function is the Fourier Transform of the PDF, this function will completely define the PDF. Think of it in the same class of functions as the function’s PDF and cumulative distribution function (CDF). Each of the 3 functions describes the random variable in a different way giving the user more insight into the features of the RV.

Pricing under Fourier – The Carr-Madan Approach

This is one of 2 popular approaches to applying Fourier Transforms to option pricing, the other being created by Lewis (2001), this approach requires numerical integration (quadrature) to solve the pricing equation, the CM approach can be priced in discrete settings as we discussed previously with discrete FT. But first we have a look at how the CM approach works before applying it so we have an understanding of FT in an option pricing setting.

First we define the payoff of a European call option where in this special case, K≡ekandST≡es. The PV of this can be defined as the following:
C0=e−rT∫k∞(eS–ek)q(s)ds
Where the standard payoff is S-K for a call option, q(s) is the risk-neutral PDF of sT
For integrability we must define c0 as eakC0 where a>0. This modified call price must be used since the call price CT(k) tends towards the asset price at time 0 as k tends toward −∞ therefore making it non-square integrable which is important when integrating with 2 functions.

Now we obtain the FT of this new variable c0 :
Ψ(v)≡∫−∞∞eivkc0dk

The inverse FT is given by:
C0=e−akπ∫0inftye−ivkΨ(v)dv

The inverse Fourier Transform is what will be used for Fast-Fourier-Transforms and option pricing.
Skipping the integration and substitution we find an expression for Ψ which is part of the call price , to see the proof see the original paper below.
ΨT(v)=e−rTϕT(v−(a+1)i)a2+a–v2+i(2a+1)v
An important note is to remember that Ψ in this case is u for the previously mentioned base-case for FT, therefore making it a combination of real an imaginary values, the fact that the unmodified call price is real means that it must be odd in its imaginary part and even in its real part, furthermore if we know the characteristic function ϕ in the first plcae we can use the inverse FT call price without solving for Ψ and obtain the unmodified call value.
The previous formulas work soley for ITM calls, next up is the inverse FT for an OTM call:
z0=1sinh(ak)12π∫−∞∞e−ivkγ(v)dv

Carr Madan

Fast Fourier Transform (FFT)

This is a algorithm designed to calculate the sum below under the CM framework I described about, this is now in discrete time as you shall see, it’s also interesting to note that this algorithm dates back to 1965 (Cooley)

w(u)=∑j=1Ne−i2πN(j−1)(u−1)x(j),u=1,…,N (1)

We can now define vk≡η(j−1) and numerically approximate our inverse ITM call price:
C0=e−akπ∫0inftye−ivkΨ(v)dv
C0≈e−akπ∑Nj=1e−ivjkΨ(vj)η

When it comes to η its worth pointing out that it can be represented in (1) as λη=2πN where lambda is representing a spacing term. When applying FFT the η is calibrated differently and is an essential term for the integration’s accuracy. Setting a small value for η we will find that the measurement distances along the function curve are farther apart meaning the strikes that we are measuring our call price against are spaced too far apart for enough of them to be close to the call price. Ideally, we want a higher value of η so that there are more strike prices in the desired region, essentially minimising lambda to a certain level. λη=2πN can now be used as a restriction for λ and η, and using Simpsons rule (which is used for approximating integrals via quadratic functions). With this all in mind we adapt our previous call price:
C0≈e−akuπ∑Nk=1e−i2πN(j−1)(u−1)eibvjΨ(vj)η3(3+(−1)j–⋄j−1),u=1,…,N
The ⋄ is the Kronecker Delta function, it equals 1 when n=0 and a value of 0 otherwise.

Now using the CM approach, along with FFT we can compute the PV of a European Call Option, I’ve gone with the Merton (1976) Jump-Diffusion Model since it has a well-known characteristic function.

# Fourier-Based Option Pricing (FFT)

import math
import numpy as np
from numpy.fft import *
from scipy import stats # For Calibration
from numpy.fft import fft

# Merton (1976) Jump-Diffusion Model  
# For a European Call Option using the Fourier Based Approach

S0 = 100.0  # initial index level
K = 95.0  # strike level
T = 1.0  # call option maturity
r = 0.05  # constant short rate
sigma = 0.2  # constant volatility of diffusion
lmbda = 1.0  # jump frequency p.a.
mu = -0.1  # expected jump size
delta = 0.05  # jump size volatility

# Merton Characteristic Equation
def mertonCF(u, x0, T, r, sigma, lmbda, mu, delta):
    omega = x0 / T + r - 0.5 * sigma ** 2 \
        - lmbda * (np.exp(mu + 0.5 * delta ** 2) - 1)
    value = np.exp((1j * u * omega - 0.5 * u ** 2 * sigma ** 2 +
                    lmbda * (np.exp(1j * u * mu -
                    u ** 2 * delta ** 2 * 0.5) - 1)) * T)
    return value
# from here one could use the Lewis (2001) approch with numerical integration
# or as we will do, follow th Carr-Madan (1999) Approach and use FFT

# Carr-Madan Approach with FFT
def fft_call(S0, K, T, r, sigma, lmbda, mu, delta):
    k = math.log(K / S0)
    x0 = math.log(S0/S0)
    g = 4  # factor to increase accuracy
    N = g * 2048
    eps = (g * 150.0) ** -1
    eta = 2 * math.pi / (N * eps)
    b = 0.5 * N * eps - k
    u = np.arange(1, N + 1, 1)
    vj = eta * (u - 1)
    # Modificatons to Ensure Integrability
    if S0 >= k:  # ITM case
        alpha = 1.5
        v = vj - (alpha + 1) * 1j
        modCF = math.exp(-r * T) * mertonCF(
            v, x0, T, r, sigma, lmbda, mu, delta) / (alpha ** 2 + alpha - vj ** 2 + 
            1j * (2 * alpha + 1) * vj)
    else:  # OTM case - this has 2 modified characteristic functions since is can be dampened twice due to its symmetry, 
           # these are weighted later on
        alpha = 1.1
        v = (vj - 1j * alpha) - 1j
        modCF1 = math.exp(-r * T) * (1 / (1 + 1j * (vj - 1j * alpha))
               - math.exp(r * T) /(1j * (vj - 1j * alpha))- 
                 mertonCF(v, x0, T, r, sigma, lmbda, mu, delta) /
                 ((vj - 1j * alpha) ** 2 - 1j * (vj - 1j * alpha)))

        v = (vj + 1j * alpha) - 1j
        modCF2 = math.exp(-r * T) * (1 / (1 + 1j * (vj + 1j * alpha)) -
                 math.exp(r * T) /(1j * (vj + 1j * alpha))- 
                 mertonCF(v, x0, T, r, sigma, lmbda, mu, delta) / 
                 ((vj + 1j * alpha) ** 2 - 1j * (vj + 1j * alpha)))

    # Fast Fourier Transform
    kron_delta = np.zeros(N, dtype=np.float) # the Kronecker Delta Function is 0 for n = 1 to N
    kron_delta[0] = 1 # function takes value 1 for n = 0
    j = np.arange(1, N + 1, 1)

    # Simpsons Rule
    simpson = (3 + (-1) ** j - kron_delta) / 3

    if S0 >= 0.95 * K:
        fftfunction = np.exp(1j * b * vj) * modCF * eta * simpson
        payoff = (fft(fftfunction)).real # calls numpy's fft functions
        call_value_pre = np.exp(-alpha * k) / math.pi * payoff

    else:
        fftfunction = (np.exp(1j * b * vj) * (modCF1 - modCF2) *
                       0.5 * eta * simpson)
        payoff = (fft(fftfunction)).real
        call_value_pre = payoff / (np.sinh(alpha * k) * math.pi)

    pos = int((k + b) / eps) # position
    call_value = call_value_pre[pos]

    return call_value * S0

print(fft_call(S0, K, T, r, sigma, lmbda, mu, delta))
